Versie 3 alpha 0.0.4

Deze versie voegt de ondersteuning toe van een planetoïde database.

Het is nu mogelijk alle bekende planetoïden tot een bepaalde magnitude op een kaart af te beelden.
Dit is mogelijk door een database systeem te gebruiken om de baan elementen van de MPCORB lijst in op te slaan zodat ze weer snel kunnen worden verkregen.

Omdat er nog geen documentatie is, geef ik hier een korte uitleg.

De eerste step is om het MPCORB-bestand in de database te laden. Als je niet alles van het bestand wilt, kun je een simpele selectie doen. Omdat omloop-parameters in tijd veranderen, is het mogelijk om versies van deze gegevens te bewaren. Het programma zal altijd de gegevens gebruiken van de meest nabije observatietijd.
Het bestand is beschikbaar via:
ftp://cfa-ftp.harvard.edu/pub/MPCORB/
Als je problemen hebt verbinding te maken, kijk dan op het midden van deze pagina:
http://cfa-www.harvard.edu/iau/mpc.html

Dan wordt een berekening van de magnitude van iedere planetoïde op de eerste dag van de maand gemaakt. Dit maakt het mogelijk om een grove selectie te doen en de te zwakke objecten verder buiten de berekening te houden voordat de gegevens voor iedere dag worden berekend.
Dit proces gaat langzaam vanwege de grote hoeveelheid te berekenen data, misschien moet je deze knop maar klikken voor je gaat lunchen.

Dit was alles voor wat betreft de data voorbereiding.

Wanneer je een nieuwe datum instelt worden de posities van alle planetoïden helderder dan de ingestelde grens magnitude berekend voor 0 uur UT. Dit kan 5 seconden duren voor een grens magnitude 18, maar wel 15 senden voor magnitude 20.
Deze positie wordt gebruikt om de planetoïden te kiezen die op de kaart getoond gaan worden. Daarna kost het nog maar een fractie van een seconde om de exacte postie voor de ingestelde tijd te berekenen.

Je kunt de zoek dialoogbox gebruiken om iedere planetoïde te zoeken in de database.
Deze planetoïde wordt tijdelijk toegevoegd aan de dagelijkse lijst, ook wanneer de huidige magnitude de grens overschrijdt. Dit is een gemakkelijke manier om een paar zwakke doelen aan de kaart toe te voegen.

Om deze nieuwe functie te gebruiken is het nodig om de MySQL versie 4 database server te installeren.
Voor de Windows gebruiker die hier nog geen beschikking over heeft is een vereenvoudigd pakket beschikbaar.
Voor Linux stel ik voor dat je de versie gebruikt die bij jouw distributie is meegeleverd. Als dit geen versie 4 is, kun je hem downloaden vanaf www.mysql.com. Om precies te zijn, het is de MySQL client die van versie 4 moet zijn, de server zelf mag een andere versie wezen.

Wanneer het programma voor de eerste keer gestart wordt, zal het vragen om de verbindingsparameters voor de database. Ik raad je aan een nieuwe database te maken door de corresponderende knop te gebruiken, dit laadt ook een voorbeeld-bestand met 1000 planetoïden.
Als je deze mogelijkheid niet wilt gebruiken, klik dan 'Anuleren' op dit scherm zodat planetoïden niet worden weergegeven.

Het berekenen van al deze gegevens kan wat problemen geven als je processor niet krachtig genoeg is. Je kunt dit oplossen door minder data in de berekening mee te nemen door het aantal objecten te beperken die je vanaf het MPCORB bestand laadt.

Ik heb op eenzelfde computer gemerkt dat de berekening bij Windows 98 twee to drie keer langer duurt dan onder Windows 2000 of Linux.
