Installation

J'ai CdC version 2.76. Que faire pour utiliser la nouvelle version 3. Que télécharger et est-ce que je dois effacer la version 2.76 en premier

Vous devez juste installer la version 3.0 pour Windows dans le même répertoire que la version 2.76. N'essayez pas de l'installer dans un autre répertoire et n'effacez rien car les deux versions peuvent être utilisée en même temps, c'est utile tant que la version 3.0 n'est pas terminée. Utilisez cdc.exe pour la version 3.0 et ciel.exe pour la 2.76.

Est-ce que les anciens catalogues de la version 2.76 sont compatibles avec CdC v3.0 beta

Oui, tout ces catalogues peuvent être utilisé. Ouvrez simplement la configuration des catalogues pour les ajouter.

Ou est-ce que je doit installer les catalogues supplémentaires sous Linux, dans quel répertoire les mettre

Si vous avez les droits root le mieux est de les mettre dans /usr/share/apps/skychart/cat avec les catalogues de base. Mais vous pouvez aussi les installer ailleur puisque vous pouvez indiquer le chemin complet dans la configuration des catalogues.

Est-ce que je peux obtenir Cartes du Ciel pour mon Macintosh

J'ai récement compilé et executé CdC sur OSX 10.4/Intel. C'est probablement aussi possible sur les versions précédentes PPC. Mais il y a encore du travail pour que ça soit facilement utilisable et je ne suis pas un utilisateur Mac, merci de me contacter si vous avez quelques notion de programmation et que vous pouvez m'aider pour ce portage.

Qu'est ce que je fais avec MySQL

MySQL était utilisé comme base de donnée par défaut jusqu'à la version alpha 7. Depuis juin 2005 il est remplacé par SQLite. Vous pouvez toujours utiliser MySQL si vous en avez besoin, par exemple pour partager la base de donnée sur un réseau local. La base de donnée est utilisée pour enregistrer les informations sur: les éléments des astéroides et comètes, les images des objets, les lieux d'observations.

Utilisation

Est que CdC V3.0 pour Linux supporte le suivit des télescopes comme la version 2.76

Oui, CdC V3.0 pour Linux utilise INDI comme interface pour les télescopes. Tout télescope supporté par INDI doit fonctionner avec CdC.

Il n'y a pas d'option “Voir” et “Suivre” dans le menu du télescope comme en version 2.76

C'est remplacé par la fonction “verrouillé sur” (le bouton avec l'ancre). Quand le télescope est connecté et qu'aucun autre objet n'est sélectionné ce bouton verrouille sur la position du télescope.

Pourquoi le ciel est-il bleu

Parce qu'il fait jour ou que la Lune est levée. Vous pouvez changer cela par le menu Configuration , Affichage, Voir la couleur du ciel.

Comment est-ce que je peux voir une carte avec la simulation des planètes avant 3000 BC

Ce n'est pas possible car nous n'avons pas de méthode de calcul fiable avant 3000 BC. Même l'éphéméride à long terme DE 406 du JPL ne va pas avant cette date.

Comment changer la langue du programme pour ma propre langue

Depuis la version beta 3.0.1.2 la langue est détectée automatiquement au lancement du programme. Le menu Configuration / Système / Langues permet d'en choisir une autre.

Est-ce que CdC V3 supporte le protocol DDE

Pas pour le moment. Le protocole standard de la version 3 est TCP/IP pour des questions de portabilité. DDE sera surement implémenté dans le future, peut-être à travers une passerelle.

Pourquoi avec Linux les caractères sont trop petit / trop gros

C'est un problème avec votre configuration Gtk1. Il y a plusieurs manière de le résoudre. Si vous utilisé KDE vous pouvez lui demander d'utiliser ses styles pour les applications Gtk.
